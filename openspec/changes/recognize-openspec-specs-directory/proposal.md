## Why

`documentation.docs-specs-exists` and `ai-context.docs-specs-exists` both only check for a `docs/specs/` directory, even though an OpenSpec repo's specs canonically live at `openspec/specs/` (as generated by `openspec sync`/`archive`). This means any OpenSpec-convention repo — including this one — is incorrectly scored as missing discoverable specs (0/10 on both metrics) despite having a fully maintained specs directory, just under a different, provider-specific path.

## What Changes

- `documentation.docs-specs-exists` and `ai-context.docs-specs-exists` SHALL also pass when `openspec/specs/` exists **and** the repo's detected provider set includes `openspec`. `docs/specs/` continues to satisfy the check on its own, regardless of detected providers.
- Add a `specsDir(ctx)` helper (mirroring the existing `adrDir(ctx)` pattern) that resolves to `docs/specs`, `openspec/specs` (only when `openspec` is detected), or `null`.
- No change to point values, category assignment, or the `provider` tag on either metric — both remain base (non-provider-scoped) metrics; only the detection logic becomes provider-aware.

## Capabilities

### New Capabilities

(none — this extends an existing capability)

### Modified Capabilities

- `readiness-metrics`: "Documentation Category Metrics" and "AI Context Category Metrics" requirements change so their `docs/specs` checks also accept `openspec/specs` when the `openspec` provider is detected.

## Impact

- `src/metrics/helpers.ts`: add a `specsDir(ctx)` helper.
- `src/metrics/documentation.ts`: `documentation.docs-specs-exists`'s `check` uses `specsDir(ctx) !== null` instead of `ctx.hasPath('docs/specs')`.
- `src/metrics/ai-context.ts`: `ai-context.docs-specs-exists`'s `check` uses the same helper.
- No change to `instruction`/`remediation` text on either metric — the fix text ("Create a docs/specs/ directory") still applies to non-OpenSpec repos; OpenSpec repos with `openspec/specs/` already satisfy the check and never see that instruction.
- No change to scoring math, provider detection, CLI flags, or any other metric.
